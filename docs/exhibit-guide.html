BCMS 名刺管理システム 展示ガイド

このシステムは、紙の名刺をデジタル化し、連絡先情報を整理・検索できるようにする
ための統合プラットフォームです。 Business Card Management System(BCMS)は、
AIを活用したOCR技術とクラウドストレージを組み合わせ、効率的な名刺管理を実現します。

技術スタック

BCMS は以下の技術で構成されています。

フロントエンド

  • Next.js 16 (App Router)
  • React 19
  • 型安全な Supabase クライアント

バックエンド

  • Next.js API Routes
  • 非同期ジョブで Python スクリプト実行
  • Supabase Storage/Database 連携

画像処理 & OCR

  • Python + OpenCV による前処理
  • YomiToku OCR + Gemini 連携
  • HEIC/HEIF 対応（pillow-heif）

ストレージ & 認証

  • Supabase Postgres
  • Supabase Storage
  • Supabase Auth (UUID 利用)

使い方ガイド

1. ダッシュボードを開く

  • ブラウザで 名刺管理ダッシュボード にアクセスします。
  • 上部には検索欄、中央に名刺一覧、右側にアップロードと活動サマリーが表示されます。
  • 各カードには、名前・所属・メール・電話・解析状況がまとめられています。

2. 名刺を検索・閲覧する

  1. 検索ボックスに「名前」「会社」「タグ」などのキーワードを入力します。
  2. 並び順を「最近更新」または「アップロード順」から選択できます。
  3. カードをクリックすると詳細ページに移動し、原本画像・前処理済み画像・抽出情報が確認できます。

3. 名刺をアップロードする

  1. 右上の「名刺をアップロード」ボタンまたはメニューから スキャン ページに移動します。
  2. 名刺を登録したいプロジェクトを選択します。
  3. 画像の追加方法は 2 通りあります。
    • ファイルの選択: JPEG / PNG / HEIC を複数枚まとめて選択できます。
    • カメラで撮影: 内蔵・外付けカメラで撮影すると、自動で縦向き補正と明るさ調整がかかり、プレビューに表示されます。
  4. 不要な画像はプレビューカードの「削除」ボタンで取り除けます。
  5. 準備ができたら 解析キューに送信 を押すと、バックエンドで前処理 → OCR → Supabase 登録まで自動で実行されます。

4. 解析結果の確認

  • 進捗はダッシュボードに戻ると自動で反映されます。必要に応じてブラウザを更新してください。
  • カードのバッジから「前処理済み」「最新解析日」などを確認できます。
  • 詳細ページでは抽出された連絡先情報（名前・役職・メール・電話・住所など）を即座に閲覧できます。

注意事項

  • 展示環境ではカメラ利用にブラウザの許可が必要です。初回アクセス時にダイアログが表示されたら「許可」を選択してください。
  • HEIC 画像は自動で JPEG に変換して取り込まれますが、変換に数秒かかる場合があります。
  • アップロード直後は解析完了まで少し時間がかかります。カードの「最新解析日」が更新されるまでお待ちください。
  • トラブルシューティングは残していませんが、問題が発生した場合はスタッフまでお知らせください。

作者からのコメント

情報学部1年の岩本です。今回、初めてシステムの構築を行いました。AIエージェントとともに対話しながらの開発なので、難しいことは意識せずにできましたが、前提知識が不足しているのが問題でした。今まで、抽象的に理解していたDB、API、フロントエンド、バックエンドなどの概念が、実際に触れることでまだまだ知識不足であるということを思い知らされました。非常に良い経験となったため、皆さまもぜひご挑戦ください。

デモでは Supabase のサービスロールキーを利用していますが、本番環境では認証ユーザー単位で権限を分離する構成を想定しています。 来場者の皆さまには、名刺情報が安全に整理されていく様子をぜひ体験していただければ幸いです。

今後の展望

  • 多言語対応: 現状は日本語・英語の項目抽出に最適化していますが、将来的には多言語 OCR と翻訳を組み合わせてグローバル来場者にも対応します。
  • フォローアップ自動化: 抽出したメールアドレスや電話番号から CRM 連携し、展示会後の自動連絡テンプレートを生成する機能を追加予定です。
  • 解析モデルの拡張: Gemini API の利用状況に応じてローカル推論モデルへの切り替えや、社内用に fine-tune したフィールド抽出モデルを導入する計画があります。
  • 権限管理の強化: Supabase Auth と RLS を組み合わせ、プロジェクト単位で閲覧・編集権限を柔軟に設定できる管理画面を整備します。
